Usage


Homepage and log in


After successful installing GreyCloud, enter the URL from your server. If everything works correct, a page with a list of links appears. The URL for this homepage is ‘/home’. To get to the administration page, click on the link ‘GreyCloud administration’. If you are not logged in, a login form appears. Enter any credentials for a valid user.

Create a new user


GreyCloud uses an intuitive user interface for managing cloud services. After you signed in, the administration page with your databases appears under ‘administration/databases’. To create a new user, click on the ‘users’ symbol to get to ‘/administration/users’, then click on the green ‘+ New’ link. A page with the URL ‘/administration/users/new’ appears. Enter a name which only contains alphanumeric chars and underscores. Enter the same password into the two password fields. Optionally you can enter an e-mail and a real name so other users can contact this user.
Please note that GreyCloud does not use permissions so every user can create new accounts or cloud services.

Create a new database


Click on the database symbol to get to ‘/adminstration/databases’. Then, click on ‘+ New’. Enter a unique name for your database. Note that you cannot edit the name of the database after creating it. Optionally you can add a description for your database.

Managing virtual files


GreyCloud has a virtual file system which stores files and folders into a database. To manage this file system, click on the link ‘Filemanagement’ or enter ‘/filemanagement/root’ in your web browser. Names for files and folders can only contain alphanumeric chars, underscores and dots. To go back to a parent folder, click ‘..’.

Create a new web service


To create a web service you have to upload your scripts into the virtual file system. Go to ‘/administration/services’ or click on the ‘services’ symbol. Then enter a name for the service and enter the path to the main script of your service.

Logfile


To see a log file with all activities, click on ‘Logfile’ or enter the URL ‘/administration/logfile’ into your browser.
