Installation and configuration


Download


This section shows how you can install GreyCloud on Linux Ubuntu with the Apache web server. Note that on other Linux distributions the directories might be different.
You can download the newest GreyCloud version from GitHub. For productive use select a stable release.

Install required components


1) Install the apache web server by running the following command:
#sudo apt-get update
#sudo apt-get install apache2

2) Then, run
#python --version
to see if the distribution uses Python ≥ 3.4.
If the version is < 3.4, install a new one.

3) Install mod WSGI and Python PIP:
#sudo apt-get install python3-pip
#sudo apt-get install libapache2-mod-wsgi-py3

4) Install Python Virtualenv:
#sudo pip3 install virtualenv

5) Create a virtual environment for Python:
#cd /path/to/venv
#virtualenv GreyCloudEnv

6) Install Django 2 into the virtual environment and your normal Python environment:
#source GreyCloudEnv/bin/activate
(GreyCloudEnv)#pip install django
(GreyCloudEnv)#deactivate
(GreyCloudEnv)#pip3 install django

Install GreyCloud


After installing required components, extract the software into ‘/var/www/html’ and rename it to ‘GreyCloud‘. You need root permissions to do that. Run the setup script by typing
#sudo python3 setup.py install
Into your command line and enter the information for an initial account.
Set the read and write permissions for the apache webserver:
#sudo chown root:www-data –R /var/www/html/GreyCloud
#sudo chmod 774 –R /var/www/html/GreyCloud
Go to ‘_conf/usersettings.con‘ and set ‘USE_APACHE‘ to true. You also have to set the path to the virtual environment in ‘VENV_PATH’ and ‘PYTHON_VERSION’.

Configure Apache


Configure Apache by editing ‘/etc/apache2/sites-available/000-default.conf. You need root permissions to do that.

000-default.conf:
<VirtualHost *:80>
    …
    Alias /static /var/www/html/GreyCloud/static
    <Directory /var/www/html/GreyCloud/static>
         Require all granted
    </Directory>
    <Directory /var/www/html/GreyCloud/GreyCloud>
         <Files wsgi.py>
            Require all granted
        </Files>
    </Directory>
    WSGIDaemonProcess GreyCloud python-home=/path/to/venv
    WSGIProcessGroup GreyCloud
    WSGIScriptAlias / /var/www/html/GreyCloud/GreyCloud/wsgi.py
</VirtualHost>

Alternative you can create a new virtual host file and enable it with
#sudo a2ensite [Name].conf
Then, restart the server with
#sudo service apache2 restart

Change settings


Open the file GreyCloud/usersettings.json in a text editor. Change the ‘ALLOWED_HOSTS’ settings if you want to allow a connection only to special hosts. If you want to use every host, add ‘*’ to the list and remove all other hosts. To use time zones, set ‘USE_TIMEZONE’ to true. Enter your time zone in ‘ZIMEZONE’. Disable ‘DEBUG’ if you want to use GreyCloud in a productive environment. Enable ‘SECURE’ if you want to use GreyCloud with HTTPS.
